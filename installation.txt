Trackmania-AI Python setup (Windows)

Odporúčané:
1. vytvor si virtual environment
2. upgradni pip
3. nainštaluj Python balíčky nižšie

Príkazy:

python -m venv .venv
.venv\Scripts\activate
python -m pip install --upgrade pip

Základné balíčky pre projekt:
python -m pip install numpy trimesh gymnasium vgamepad matplotlib inputs "pyglet<2"

PyTorch:

Ak chceš CUDA verziu, použi oficiálny PyTorch index-url pre svoju GPU/CUDA kombináciu.
Príklad pre CUDA 12.8:
python -m pip install torch --index-url https://download.pytorch.org/whl/cu128

Ak chceš CPU-only verziu:
python -m pip install torch --index-url https://download.pytorch.org/whl/cpu

Poznámky:

- `vgamepad` na Windows potrebuje nainštalovaný ViGEmBus driver.
- pre vizualizáciu cez `Vizualizer.py` je potrebný `pyglet`
- projekt ďalej potrebuje:
  - Trackmania
  - OpenPlanet plugin, ktorý posiela live data do Pythonu
  - exportované map/block súbory v priečinkoch projektu

Oficiálne odkazy:
- PyTorch install selector: https://pytorch.org/get-started/locally/
- vgamepad: https://pypi.org/project/vgamepad/
- ViGEmBus driver: https://github.com/ViGEm/ViGEmBus/releases/

Riešime odhad AABB boxu auta z Trackmanie. Po novom Openplanet exporte som pozrel `OpenplanetNext\Extract\GameData\Vehicles`. Export už obsahuje `CarSport.Item.Gbx`, `PhyModelSport.VehiclePhyModel.Gbx`, `VisModelSport.VehicleVisModel.Gbx`, tunings, materiály atď., ale pre Stadium/CarSport som stále nenašiel samostatný `CarSport.Shape.Gbx`. `PhyModelSport` tam je, ale náš parser zatiaľ nevie dekódovať chunk `0x090EA003`, takže oficiálny fyzikálny hitbox stále nemáme.

Spravil som preto dva približné AABB odhady:

1. Z reálne exportovaného vizuálneho meshu `MainBody.Mesh.gbx`, exportnutého do OBJ:
   - `carExport\CarSport_Stadium_Standard\MainBody_*_lod1.obj`
   - AABB size približne: `[2.1432, 3.7842, 0.8822]`
   - center približne: `[0.0004, -0.2737, 0.4292]`

2. Z pridaného ManiaPark skinu `car_primitives\MainBodyHigh.Solid.Gbx`.
   Tam som ručne vyčítal starý `CPlugSolid` formát, 5 visual sekcií + transformácie kolies.
   - OBJ axes min: `[-1.0663, -1.9345, -1.1691]`
   - OBJ axes max: `[1.0663, 2.1447, 0.0101]`
   - AABB size približne: `[2.1326, 4.0792, 1.1792]`
   - center približne: `[0.0000, 0.1051, -0.5795]`
   - half-extents približne: `[1.066, 2.040, 0.590]`

Dôležité: oba výsledky sú len odhady, nie potvrdený oficiálny physics hitbox. Druhý odhad z `car_primitives` vyzerá použiteľnejšie ako konzervatívny AABB auta, lebo má väčšiu dĺžku aj výšku a zahŕňa primitívnejší/kolíznejšie vyzerajúci model. Na ďalšie použitie by som teda zatiaľ bral približne `size = [2.13, 4.08, 1.18]`, kým sa nepodarí dekódovať priamo `PhyModelSport.VehiclePhyModel.Gbx`.
